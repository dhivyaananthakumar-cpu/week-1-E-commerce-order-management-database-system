 E-Commerce Order Management Database System

 Week 1 - Requirement Analysis

 Project Description
The E-Commerce Order Management Database System is designed to manage the daily operations of an online shopping platform. It maintains information about customers, products, categories, suppliers, orders, payments, shipments, and customer reviews.

 Objective
- Manage customer details.
- Maintain product and category information.
- Process customer orders.
- Track payments and shipments.
- Store customer reviews.

 Features
- Customer Management
- Product Management
- Category Management
- Supplier Management
- Order Management
- Payment Management
- Shipment Tracking
- Review Management

 Project Scope
This project focuses on analyzing business requirements and designing the database structure for an e-commerce order management system.

 Technologies
- Database Design
- ER Diagram
- Requirement Analysis
- Business Requirement Document (BRD)
- Software Requirement Specification (SRS)

 Week 2 - Entity and Relationship Analysis

 Project Title
   E-Commerce Order Management Database System

 Project Overview
This project focuses on analyzing the entities and relationships required for the E-Commerce Order Management Database System. Based on the Software Requirement Specification (SRS) prepared in Week 1, the entities, attributes, primary keys, foreign keys, constraints, and relationships were identified to support the database design.

 Objectives
- Identify all entities in the system.
- Define attributes for each entity.
- Identify Primary Keys (PK) and Foreign Keys (FK).
- Apply database constraints such as NOT NULL, UNIQUE, DEFAULT, and CHECK.
- Analyze relationships between entities.
- Define the cardinality of each relationship.

 Deliverables
- Entity Analysis Report
- Entity Relationship Analysis Report
- Entity Identification Report
- Entity Attribute List
- Primary Key and Foreign Key Identification
- Relationship Analysis
- Cardinality Analysis
